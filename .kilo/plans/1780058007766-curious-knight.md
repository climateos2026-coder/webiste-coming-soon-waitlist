# Vercel Deployment Guide for ClimateOS 2026 Website

## Project Overview
- **Framework**: Next.js 16 (App Router) + TypeScript
- **Styling**: Tailwind CSS 4
- **Output**: Static-rendered pages with optional server-side features
- **Compatible**: Node.js 18.x or 20.x

---

## Step 1: Prerequisites

### 1.1 Git Repository Setup
Ensure your code is in a Git repository:
```bash
git init
git add .
git commit -m "Ready for Vercel deployment"
```

### 1.2 Vercel Account
- Sign up at [vercel.com](https://vercel.com) or use `vercel login` CLI
- Install Vercel CLI globally: `npm install -g vercel`

---

## Step 2: Deploy via Vercel Dashboard (Recommended)

### 2.1 Import Project
1. Go to [vercel.com/new](https://vercel.com/new)
2. Connect your Git provider (GitHub, GitLab, Bitbucket)
3. Select your repository
4. Vercel auto-detects Next.js framework

### 2.2 Configure Project Settings
**Project Name**: `climate-os-26` (or your preferred name)

**Framework Preset**: Next.js (auto-detected)

**Root Directory**: `.` (leave empty if repo root)

**Node.js Version**: 18.x or 20.x (set in Settings → General → Node.js Version)

### 2.3 Existing vercel.json
The project already contains `vercel.json` with configured redirects and rewrites - no changes needed.

---

## Step 3: Environment Variables Configuration (Optional)

If using interactive features later (waitlist form), these variables may be needed:

| Name | Value | Notes |
|------|-------|-------|
| `NEXT_PUBLIC_SUPABASE_URL` | `https://[project].supabase.co` | Required for database |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | From Supabase Settings → API | Required for database |
| `SUPABASE_SERVICE_ROLE_KEY` | From Supabase Settings → API | Server-side only |
| `BREVO_API_KEY` | From Brevo account | Welcome email sending |
| `BREVO_FROM_EMAIL` | Verified sender email | Email "from" address |
| `BREVO_FROM_NAME` | `ClimateOS Team` | Email sender name |
| `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` | Your domain | Analytics tracking |
| `NEXT_PUBLIC_PLAUSIBLE_URL` | Plausible server URL | Analytics endpoint |
| `NEXT_PUBLIC_EVENT_PHASE` | `waitlist` | Content display mode |

Variables prefixed with `NEXT_PUBLIC_` are exposed to browser. Skip all for static-only deployment.

## Step 4: Build Configuration

### 4.1 Build Settings (Vercel Dashboard → Settings → Build)

| Setting | Value |
|---------|-------|
| **Build Command** | `npm run build` |
| **Output Directory** | `.next` (auto-detected) |
| **Install Command** | `npm install` |
| **Node.js Version** | 18.x or 20.x (LTS) |

### 4.2 vercel.json Configuration
The project includes `vercel.json` with configured redirects and rewrites:
- `/waitlist` → `/` (rewrite)
- `/tracks/:path*` → `/#tracks` (301 redirect)
- No trailing slash

No changes needed - Vercel applies these automatically.

## Step 5: Deployment Verification

### 5.1 Pre-deployment Checklist
- [ ] `npm run build` works locally
- [ ] `npm run type-check` passes

### 5.2 Trigger Deployment
```bash
git push origin main
# or use Vercel CLI
vercel --prod
```

---

## Step 6: Troubleshooting Common Errors

### 6.1 Build Failures
**Error**: TypeScript type errors during build
- **Fix**: Run `npm run type-check` locally first, ensure all types are correct

### 6.2 Security Headers
**Error**: Mixed content warnings
- **Fix**: All URLs in env vars must use `https://`

**Error**: Images not loading
- **Fix**: Image domains already configured in `next.config.ts`

### 6.3 Performance
- Vercel automatically caches node_modules between builds

---

## Step 7: Post-Deployment

### 7.1 Custom Domain (Optional)
1. Vercel Dashboard → Domains → Add
2. Configure DNS records as instructed
3. Update CSP in `next.config.ts` if needed

### 7.2 Monitoring
- Check Vercel Analytics for performance
- Enable Provenance for supply chain security
- Set up uptime monitoring

### 7.3 Environment Updates
To update environment variables:
1. Vercel Dashboard → Settings → Environment Variables
2. Edit or add variables
3. Redeploy to apply changes

---

## Step 8: CLI Deployment (Alternative)

```bash
# Login to Vercel
vercel login

# Deploy preview
vercel

# Deploy to production
vercel --prod

# Link existing project
vercel link
```

---

## Step 9: Security Considerations

### 9.1 CSP Headers
The project includes Content Security Policy headers in `next.config.ts` - update if using custom domains.

### 9.2 HTTPS Requirements
All external URLs (Supabase, Brevo, Plausible) must use HTTPS in production.

### 9.3 Secrets Management
- Never expose server-side secrets in client code
- Use Vercel's Environment Variables UI (not CLI) for sensitive values
